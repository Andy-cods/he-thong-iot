### 1.1 Tokens bổ sung (thêm vào `tailwind.config.ts`)

V1.1 mở rộng palette theo 2 hướng: (a) 10 màu state cho BOM snapshot line (không thêm màu mới, dùng gradient từ palette có sẵn); (b) màu neutral thêm cho tree hierarchy.

```ts
// tailwind.config.ts — diff ADD (không xoá gì)
colors: {
  // ... existing V1.0 (slate, brand, cta, success, warning, danger, info, zebra)

  // V1.1 MỚI: tree hierarchy depth indicators
  tree: {
    line:     "#CBD5E1",   // slate-300 — đường kẻ dọc connector
    dot:      "#94A3B8",   // slate-400 — node dot collapsed
    highlight:"#FEF3C7",   // warning-soft — row đang edit inline
    dropzone: "#DBEAFE",   // info-soft — drag-over target
  },

  // V1.1 MỚI: 10-state palette cho snapshot line (dẫn xuất từ token V1.0)
  state: {
    planned:       "#E2E8F0", // slate-200 (background)
    planned_ink:   "#334155", // slate-600 (text)
    purchasing:    "#DBEAFE", // info-soft (bg)
    purchasing_ink:"#0369A1", // info (text)
    production:    "#FEF3C7", // warning-soft (bg)
    production_ink:"#9A3412", // cta-press (text, safety orange darkest)
    inbound_qc:    "#DBEAFE", // info-soft (bg, dashed border để phân biệt với purchasing)
    inbound_qc_ink:"#075985", // info-strong (text)
    prod_qc:       "#FEF3C7", // warning-soft (bg, dashed border)
    prod_qc_ink:   "#78350F", // warning-strong (text)
```

**Giải thích lựa chọn:**
- 10 state dùng **gradient xanh** (purchasing → available → reserved → assembled) để thể hiện tiến trình "càng xanh càng gần hoàn tất"; `issued` là slate đậm vì đây là trạng thái trung gian (đã xuất khỏi kho nhưng chưa lắp); `closed` xám vì đã đóng lifecycle.
- Mọi màu đều đã trong palette V1.0 — KHÔNG thêm hex mới ngoài 2 tint xanh đậm cho `reserved` và `assembled` (derive từ `success` base).
- Contrast check: mọi cặp bg/ink đều ≥ 4.5:1 (WCAG AA body text).

**Data flow:**
- TanStack Query keys:
  - `['bom-templates', { q, category, status, page }]` — list query, `staleTime: 60s`.
  - `['bom-templates', 'count']` — total count cho subtitle.
- nuqs URL state: `?q=...&category=...&status=...&page=1`.
- Form state: `react-hook-form` + Zod schema `BomTemplateCreateSchema`.
- Mutation: `POST /api/bom-templates` → invalidate `['bom-templates']` on success + navigate sang detail page.

**Interactions state machine:**
| Event | State | Next state | Action |
|---|---|---|---|
| page load | `loading` | `populated` (nếu có data) hoặc `empty` | fetch list |
| type in search | `populated` | `populated` | debounce 300ms → refetch |
| click `+ Tạo` | `populated` | `sheet-open` | open CreateTemplateSheet |
| sheet submit valid | `sheet-open` | `creating` | POST, button spinner |
| server 200 | `creating` | `redirect` | navigate `/bom/templates/[newId]`, toast success |
| server 400 | `creating` | `sheet-open + error` | inline field error |
| row click | `populated` | `redirect` | navigate detail |
| fetch error | `loading` | `error` | show ErrorBanner, retry button |

**Keyboard shortcuts:**
- `N` — mở sheet tạo mới (khi không focus input).
- `/` — focus search input.
- `Enter` (row focus via Tab) — navigate detail.
- `Esc` — close sheet / clear search.
- `Ctrl+K` — CommandPalette (reuse V1.0).

**Acceptance criteria (8 điểm):**
1. [ ] List render đúng 20 rows/page, virtualized khi > 200 templates.
2. [ ] Search debounce 300ms, tìm được theo mã hoặc tên (normalize bỏ dấu Việt).
3. [ ] Filter status `ALL/ACTIVE/HAS_DRAFT` hoạt động (nuqs param persist qua reload).
4. [ ] Empty state hiển thị đúng 2 variant: first-time vs filter-no-match.
5. [ ] Sheet tạo mới validate Zod: code regex `^BOM-[A-Z0-9]+-[0-9]{3}$`, name required 3-255 chars.
6. [ ] Sau tạo thành công, navigate đúng detail page với URL `/bom/templates/[uuid]`.
7. [ ] Keyboard `N` mở sheet, `Esc` đóng, `/` focus search.
8. [ ] A11y: axe-core 0 serious violation, focus ring 3px `info` color, skip-to-content link hoạt động.

**Aggregation logic (note cho dev — không visual):**
```sql
-- Multi-level rollup per item trong 1 order
SELECT
  item_id,
  SUM(gross_required_qty) AS total_required,
  SUM(received_qty + reserved_qty) AS covered,
  SUM(remaining_short_qty) AS shortage,
  MIN(eta_date) AS eta_nearest
FROM sales.bom_snapshot_line
WHERE customer_order_id = :orderId
  AND line_status IN ('PLANNED','PURCHASING','INBOUND_QC')
GROUP BY item_id
HAVING SUM(remaining_short_qty) > 0
ORDER BY SUM(remaining_short_qty) DESC;
```

### 4.4 State machine implementation (pseudo-code)

```ts
// apps/web/src/lib/state-machines/bom-snapshot-line.ts

export type SnapshotLineStatus =
  | 'PLANNED' | 'PURCHASING' | 'IN_PRODUCTION'
  | 'INBOUND_QC' | 'PROD_QC' | 'AVAILABLE'
  | 'RESERVED' | 'ISSUED' | 'ASSEMBLED' | 'CLOSED';

const VALID_TRANSITIONS: Record<SnapshotLineStatus, SnapshotLineStatus[]> = {
  PLANNED:       ['PURCHASING', 'IN_PRODUCTION', 'CLOSED'],
  PURCHASING:    ['INBOUND_QC', 'CLOSED'],
  IN_PRODUCTION: ['PROD_QC', 'CLOSED'],
  INBOUND_QC:    ['AVAILABLE', 'PURCHASING'],       // QC PASS hoặc FAIL
  PROD_QC:       ['AVAILABLE', 'IN_PRODUCTION'],    // QC PASS hoặc rework
  AVAILABLE:     ['RESERVED', 'CLOSED'],
  RESERVED:      ['ISSUED', 'AVAILABLE'],           // ISSUED hoặc release reservation
  ISSUED:        ['ASSEMBLED'],
  ASSEMBLED:     ['CLOSED'],
  CLOSED:        [],                                // terminal
};

export function isValidTransition(
  from: SnapshotLineStatus,
  to: SnapshotLineStatus,
): boolean {
  return VALID_TRANSITIONS[from]?.includes(to) ?? false;
}

export function getAllowedTransitions(
  from: SnapshotLineStatus,
): SnapshotLineStatus[] {
  return VALID_TRANSITIONS[from] ?? [];
}

// Business logic: transition with audit
export async function transitionSnapshotLine(
  lineId: string,
  toStatus: SnapshotLineStatus,
  actor: { id: string; name: string },
  reason: string,
  reference?: { type: string; id: string },
): Promise<{ ok: true } | { ok: false; error: string }> {
  const line = await db.query.bomSnapshotLine.findFirst({ where: eq(id, lineId) });
  if (!line) return { ok: false, error: 'Line not found' };
  if (!isValidTransition(line.lineStatus as SnapshotLineStatus, toStatus)) {
    return { ok: false, error: `Transition ${line.lineStatus} → ${toStatus} not allowed` };
  }
  await db.transaction(async (tx) => {
    await tx.update(bomSnapshotLine)
      .set({ lineStatus: toStatus, updatedAt: new Date() })
      .where(eq(id, lineId));
    await tx.insert(stateTransitionLog).values({
      entityType: 'bom_snapshot_line',
      entityId: lineId,
      fromStatus: line.lineStatus,
      toStatus,
      actorId: actor.id,
      actorName: actor.name,
      reason,
      referenceType: reference?.type,
      referenceId: reference?.id,
      timestamp: new Date(),
    });
  });
  return { ok: true };
}
```

## §5 Partial Qty Model — 9 Cột UI Representation

Đây là design pattern cốt lõi cho Snapshot Board (§2.5.B). R2:247 yêu cầu 9 trường qty song song thay vì boolean yes/no. UI phải truyền tải đúng.

### 5.1 Bảng header 9 cột

| # | Cột | Column name DB | Formula | Tooltip mô tả |
|---|---|---|---|---|
| 1 | Yêu cầu | `net_required_qty` | `qty_per_parent × parent_qty` (recursive) | "Số lượng net yêu cầu, chưa tính scrap" |
| 2 | Brutal (Gross) | `gross_required_qty` | `net_required_qty × (1 + scrap_rate)` | "Số lượng gross sau khi cộng scrap %" |
| 3 | Mở mua | `open_purchase_qty` | `SUM(po_line.ordered_qty WHERE linked_snapshot_line_id = X AND status IN ('SENT','CONFIRMED','PARTIAL_RECEIVED'))` | "Tổng qty đã đặt mua nhưng chưa nhận đủ" |
| 4 | Đã nhận | `received_qty` | `SUM(stock_transaction.qty WHERE tx_type='RECEIPT' AND ref=X)` | "Số lượng đã receipt về kho (không check QC)" |
| 5 | QC đạt | `qc_pass_qty` | `SUM(qc_inspection.pass_qty WHERE snapshot_line=X)` | "Số lượng đã QC PASS, có thể reserve" |
| 6 | Reserved | `reserved_qty` | `SUM(stock_reservation.qty WHERE snapshot_line_id=X AND released_at IS NULL)` | "Số lượng đã khoá cho line này (chưa xuất)" |
| 7 | Đã cấp | `issued_qty` | `SUM(stock_transaction.qty WHERE tx_type='ISSUE' AND ref=X)` | "Đã xuất kho tới assembly (chưa lắp)" |
| 8 | Đã lắp | `assembled_qty` | `SUM(stock_transaction.qty WHERE tx_type='ASM_CONSUME' AND ref=X)` | "Đã tiêu hao vào FG serial" |
| 9 | Còn thiếu | `remaining_short_qty` | `gross_required - (received_qty + open_purchase_qty)` hoặc generated column | "Tổng còn thiếu = (gross) - (đã mở mua + đã nhận) — âm = over-received" |

**Generated column approach (PostgreSQL):**
```sql
remaining_short_qty numeric(18,6) GENERATED ALWAYS AS (
  GREATEST(0, gross_required_qty - (received_qty + open_purchase_qty))
) STORED;
```

Tuỳ chọn: không dùng GREATEST để cho phép âm (over-received) — design team quyết định.
