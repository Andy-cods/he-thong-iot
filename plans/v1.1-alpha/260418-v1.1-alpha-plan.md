### 2.3 `0003b_bom_tables.sql` — tables + Drizzle rewrite

**QUAN TRỌNG (R1):** schema Drizzle `packages/db/src/schema/bom.ts` hiện khai báo 3 bảng:

- `bom_template` (code, productItemId, name, description, isActive)
- `bom_revision` (templateId, revCode, status, releasedAt, ...)

V1.1-alpha **loại bỏ `bom_revision`** (defer V1.1-full) và đổi `bom_line` thành **self-referencing tree** (`parent_line_id` thay vì `parent_item_id`/`child_item_id` flat). Migration 0003b thay schema:

```sql
  RETURNS trigger AS $$
```

### 3.1 BOM templates (7 endpoints)

#### `GET /api/bom/templates` — list + filter

- **File:** `apps/web/src/app/api/bom/templates/route.ts`
- **Query Zod:** `{ q?: string, status?: 'DRAFT'|'ACTIVE'|'OBSOLETE', hasComponents?: 'true'|'false', sort?: 'updatedAt'|'code'|'name', sortDir?: 'asc'|'desc', page?: number(default 1), pageSize?: number(default 50, max 200) }`
- **Response:** `{ rows: BomTemplateRow[], total: number, page, pageSize }` với `BomTemplateRow = { id, code, name, componentCount, targetQty, status, parentItemSku, parentItemName, updatedAt }`
- **Business:** subquery count `bom_line.template_id=rows.id`. Filter `q` dùng `ILIKE %q%` trên code + `f_unaccent(name) ILIKE %unaccent(q)%`. Paginate server-side.
- **Role:** any logged-in user.

#### `GET /api/bom/templates/[id]` — detail + tree flatten

- **File:** `apps/web/src/app/api/bom/templates/[id]/route.ts` method `GET`
- **Response:** `{ template: BomTemplate, tree: FlatTreeNode[] }` với `FlatTreeNode = { id, parentLineId, componentItemId, componentSku, componentName, level, position, qtyPerParent, scrapPercent, uom, description, supplierItemCode, childCount, metadata }`
- **Query:** recursive CTE depth ≤ 5, ORDER BY level, position
  ```sql
  WITH RECURSIVE tree AS (
    SELECT l.*, 1::int AS depth
    FROM app.bom_line l
    WHERE l.template_id = $1 AND l.parent_line_id IS NULL
    UNION ALL
    SELECT l.*, t.depth + 1
    FROM app.bom_line l INNER JOIN tree t ON l.parent_line_id = t.id
    WHERE t.depth < 5
  )
  SELECT t.*, i.sku, i.name AS item_name,
    (SELECT count(*) FROM app.bom_line c WHERE c.parent_line_id = t.id) AS child_count
  FROM tree t LEFT JOIN app.item i ON i.id = t.component_item_id
  ORDER BY depth, position;
  ```
- **Role:** any logged-in user.

#### `POST /api/bom/templates` — create

- **Body Zod:**
  ```ts
  z.object({
    code: z.string().min(1).max(64).regex(/^[A-Z0-9_-]+$/),
    name: z.string().min(1).max(255),
    description: z.string().max(2000).optional(),
    parentItemId: z.string().uuid().optional(),
    targetQty: z.number().positive().default(1),

  ```
- **Business:** insert DRAFT, auto set `created_by` from session. Audit `auditEvent.action=CREATE, objectType=bom_template`.
- **Response 201:** `{ id, code, ... }`
- **Role:** any logged-in user (Q5).
- **Error:** 409 `{ code: 'BOM_CODE_DUPLICATE' }` nếu code trùng.

#### `PATCH /api/bom/templates/[id]` — edit metadata

- **Body Zod:** subset create (không đụng components), cho phép sửa `name, description, parentItemId, targetQty, status`.
- **Business:** transition status: DRAFT→ACTIVE OK any user, ACTIVE→OBSOLETE OK admin only (V1.1-alpha trim), OBSOLETE→* reject.
- **Audit:** `UPDATE` với `before_json`/`after_json`.

#### `DELETE /api/bom/templates/[id]` — soft delete

- **Business:** set `status='OBSOLETE'` (không xoá thực). Nếu đã OBSOLETE → 400 `ALREADY_OBSOLETE`.
- **Role:** `admin` only (per R5 brainstorm — ngăn user tự xoá BOM production).

- **Body Zod:** `{ newCode: z.string().min(1).max(64), newName?: z.string() }`
- **Business:** SELECT template + all lines → INSERT new template + lines mapping `parent_line_id` qua UUID map.
- **Role:** any logged-in user.

#### `GET /api/bom/templates/check-code` — validate unique real-time

### 3.2 BOM lines (5 endpoints)

#### `POST /api/bom/templates/[id]/lines` — add component

- **Body Zod:**
  ```ts
  z.object({
    parentLineId: z.string().uuid().nullable(),
    componentItemId: z.string().uuid(),
    qtyPerParent: z.number().positive(),
    scrapPercent: z.number().min(0).max(100).default(0),
    position: z.number().int().optional(),  // auto max+1 nếu không có
    uom: z.string().max(32).optional(),
    description: z.string().max(2000).optional(),
    supplierItemCode: z.string().max(128).optional(),
  })
  ```
- **Business:**
  - Nếu `parentLineId=null` → `level=1`
  - Nếu có → SELECT parent line.level, set `level=parent.level+1`, reject nếu > 5 (422 `MAX_DEPTH_EXCEEDED`)
  - `position` auto `MAX(position)+1` trong cùng parent
  - Validate `componentItemId` tồn tại + active
  - Reject cycle: `componentItemId` không được trùng với bất kỳ ancestor `componentItemId` (walk up parent_line chain)
- **Audit:** CREATE

#### `PATCH /api/bom/templates/[id]/lines/[lid]` — edit

- **Body Zod:** partial `{ qtyPerParent?, scrapPercent?, description?, uom?, supplierItemCode? }`
- **Business:** không cho đổi `componentItemId` (nếu đổi → delete + create mới). Audit UPDATE.

#### `DELETE /api/bom/templates/[id]/lines/[lid]?cascade=true` — delete

- **Query Zod:** `{ cascade: z.enum(['true','false']).default('false') }`
- **Business:**
  - Đếm descendants (recursive CTE)
  - Nếu count > 0 và `cascade=false` → 409 `{ code: 'HAS_CHILDREN', descendantCount: N }`
  - Nếu `cascade=true` → FK cascade tự xoá (`ON DELETE CASCADE` 0003b)
- **Audit:** DELETE (ghi 1 row cho line gốc + N row cho descendants — loop trong service).

#### `POST /api/bom/templates/[id]/lines/[lid]/move` — reorder / move

- **Body Zod:** `z.object({ newParentLineId: z.string().uuid().nullable(), newPosition: z.number().int().min(1) })`
- **Business:**
  - Validate `newParentLineId` không phải descendant của `[lid]` (cycle check recursive)
  - Validate mới level ≤ 5 (new_level = parent.level+1 hoặc 1 nếu null)
  - Update `parent_line_id`, `level` (+ recursive update level cho tất cả descendants: `new_level = old_level + (lid.new_level - lid.old_level)`)
  - Shift position các sibling
- **Complexity:** cook thành 1 stored procedure hoặc transaction TypeScript. Test kỹ.

#### `GET /api/bom/templates/[id]/tree` — flatten tree (alias detail nhưng chỉ tree, cho refresh nhanh)

- **Response:** chỉ `{ tree: FlatTreeNode[] }` — tái dụng query detail, FE dùng sau mutation để refetch nhẹ.

## §6 Test strategy

### 6.1 Unit (Vitest)

- `bom-parser.test.ts`: 4 sheet file sample, verify header row 2 detection, 7 field mapping, row count khớp.
- `bomLines.test.ts`:
  - Insert line level 1 → level 5 OK, level 6 reject
  - Cycle detect: A → B → C, thử add A as child of C → reject
  - Move line: A parent B, move A to child of B → reject (cycle)
  - Cascade delete: A → B → C → D, delete B → 3 row deleted
- `bom.ts` schema: Zod validation code regex, status enum, qty positive

### 6.2 Integration (Playwright — 4 spec)

1. **`bom-create-flow.spec.ts`** — Login → `/bom` empty → click "Tạo BOM" → fill form → submit → redirect `/bom/[id]` → add 3 component (flat) → add 1 sub-component dưới component 1 → verify tree 2 level → edit qty inline → reload → value persist → delete leaf → row gone.
2. **`bom-import-excel.spec.ts`** — Login → `/bom/import` → upload file `docs/samples/20260318_BOM_trien_khai...xlsx` → bước 2 verify 4 sheet hiện với count đúng → chọn 2 sheet → bước 3 auto-map 5 field → bước 4 preview 10 row + enable auto-create → commit → poll status → stats hiện "2 templates, ~220 lines, N items auto-created" → click template → `/bom/[id]` render tree.
3. **`admin-user-flow.spec.ts`** — Login admin → `/admin/users` list 1 user → tạo user `planner1` role planner → logout → login `planner1` → sidebar không thấy "Quản trị" → visit `/admin` direct → redirect `/` → back admin → `/admin/settings` đổi password → logout → login password mới → OK.
4. **`receiving-sync.spec.ts`** — Login warehouse → `/pwa/receive/demo` → scan 3 SKU → dialog confirm qty → toast "Đã ghi hàng đợi" (offline giả lập qua DevTools) → online lại → replay auto → verify `GET /api/admin/audit` có 3 row receipt_event.

```bash
ssh -i ~/.ssh/iot_vps root@123.30.48.215

git checkout feat/v1.1-alpha-bom-admin-receiving

# Verify tables
docker exec -i iot_postgres psql -U hethong_app -d hethong_iot -c "\dt app.bom_*"
docker exec -i iot_postgres psql -U hethong_app -d hethong_iot -c "\dt app.receipt_*"
docker exec -i iot_postgres psql -U hethong_app -d hethong_iot -c "SELECT count(*) FROM app.bom_template;"
```

### 7.3 Tag V2 backup + Docker rebuild

```bash
# Tag v2.0.0 snapshot trước khi deploy
git tag -a v2.0.0-backup-$(date +%Y%m%d) -m "Backup V2 before V1.1-alpha deploy"

# Pull + rebuild (30-45 phút trên 2 vCPU)
docker compose -f docker-compose.prod.yml build web worker
docker compose -f docker-compose.prod.yml up -d web worker
```

  - Migration 0003 **không rollback** (tables để lại empty OK — không ảnh hưởng V2)
