## Phase B — Receiving ETA Bar (ước tính 2–3 giờ)

### B1. Component `EtaProgressBar.tsx`

**File tạo mới:** `apps/web/src/components/receiving/EtaProgressBar.tsx`

```tsx
"use client";

import { differenceInCalendarDays, parseISO } from "date-fns";
import { AlertTriangle, Clock, CheckCircle2 } from "lucide-react";
import { cn } from "@/lib/utils";

export interface EtaProgressBarProps {
  etaDate: string | null;          // ISO date "YYYY-MM-DD" hoặc null
  orderedQty: number;
  receivedQty: number;
  today?: Date;                    // injectable for testing
}

export function EtaProgressBar({
  etaDate,
  orderedQty,
  receivedQty,
  today = new Date(),
}: EtaProgressBarProps) {
  // Progress bar nhận hàng
  const receivePercent =
    orderedQty > 0
      ? Math.min(100, Math.round((receivedQty / orderedQty) * 100))
      : 0;

  const isComplete = receivedQty >= orderedQty;

  // ETA logic
  let daysLeft: number | null = null;
  let etaStatus: "ok" | "warning" | "critical" | "overdue" | "no-eta" = "no-eta";
  let etaLabel = "Chưa có ETA";

  if (etaDate) {
    const eta = parseISO(etaDate);
    daysLeft = differenceInCalendarDays(eta, today);
    if (isComplete) {
      etaStatus = "ok";
      etaLabel = "Đã nhận đủ";
    } else if (daysLeft < 0) {
      etaStatus = "overdue";
      etaLabel = `Quá hạn ${Math.abs(daysLeft)} ngày`;
    } else if (daysLeft === 0) {
      etaStatus = "critical";
      etaLabel = "Hôm nay là ETA";
    } else if (daysLeft <= 2) {
      etaStatus = "critical";
      etaLabel = `Còn ${daysLeft} ngày`;
    } else if (daysLeft <= 7) {
      etaStatus = "warning";
      etaLabel = `Còn ${daysLeft} ngày`;
    } else {
      etaStatus = "ok";
      etaLabel = `Còn ${daysLeft} ngày`;
    }
  }

  const barColor = {
    ok:       "bg-emerald-500",
    warning:  "bg-amber-500",
    critical: "bg-orange-500",
    overdue:  "bg-red-500",
    "no-eta": "bg-zinc-300",
  }[etaStatus];

  const textColor = {
    ok:       "text-emerald-700",
    warning:  "text-amber-700",
    critical: "text-orange-700",
    overdue:  "text-red-700",
    "no-eta": "text-zinc-500",
  }[etaStatus];

  const Icon =
    isComplete
      ? CheckCircle2
      : etaStatus === "overdue" || etaStatus === "critical"
        ? AlertTriangle
        : Clock;

  return (
    <div className="flex flex-col gap-1">
      {/* Progress bar */}
      <div className="flex items-center gap-2">
        <div className="h-1.5 flex-1 overflow-hidden rounded-full bg-zinc-100">
          <div
            className={cn("h-full rounded-full transition-all duration-300", barColor)}
            style={{ width: `${receivePercent}%` }}
            role="progressbar"
            aria-valuenow={receivePercent}
            aria-valuemin={0}
            aria-valuemax={100}
            aria-label={`Đã nhận ${receivePercent}%`}
          />
        </div>
        <span className="w-10 shrink-0 text-right font-mono text-xs tabular-nums text-zinc-600">
          {receivePercent}%
        </span>
      </div>

      {/* ETA label */}
      <div className={cn("flex items-center gap-1 text-xs", textColor)}>
        <Icon className="h-3 w-3 shrink-0" aria-hidden="true" />
        <span>{etaLabel}</span>
        {etaDate && !isComplete && (
          <span className="text-zinc-400">· ETA {etaDate}</span>
        )}
      </div>
    </div>
  );
}
```

## Phase D — BOM Progress per Order (ước tính 4–6 giờ)

### D1. Phân tích quan hệ schema

**Luồng data:** `bom_template` → (explode via `bom_revision` + `bomSnapshotLine`) → `sales_order`

**Vấn đề thực tế:** `bom_snapshot_line` join với `sales_order` qua `order_id`. Để hiển thị progress per BOM line, cần:
1. Biết `templateId` của BOM đang xem
2. Load các `sales_order` có `bom_template_id = templateId` VÀ đã có snapshot lines
3. Khi chọn 1 order, query `bom_snapshot_line` theo `order_id` và group theo `component_item_id`

**Cảnh báo schema:** `bom_snapshot_line` không có field `bom_line_id` (không link trực tiếp về `bom_line`). Link chỉ qua: `bom_snapshot_line.revision_id` → `bom_revision.template_id` → `bom_template.id`. Và `component_item_id` khớp với `bom_line.component_item_id`. **Không có 1-1 mapping tuyệt đối** nếu BOM có 2 line cùng item.

**Giải pháp Phase D V1.5:** Dùng `component_item_id` làm join key (chấp nhận edge case 2 line cùng item). API sẽ aggregate theo `component_item_id` từ snapshot, join với BOM tree để hiển thị status per tree row.

### D2. API Endpoint

**File tạo mới:** `apps/web/src/app/api/bom/templates/[id]/order-status/route.ts`

**Signature:**
- `GET /api/bom/templates/[id]/order-status?orderId=<uuid>`
- Response: `{ lines: OrderStatusLine[], orderInfo: {...} }`

```typescript
// OrderStatusLine
interface OrderStatusLine {
  componentItemId: string;
  componentSku: string;
  // Từ bom_snapshot_line aggregate:
  grossRequiredQty: number;
  openPurchaseQty: number;
  receivedQty: number;
  qcPassQty: number;
  reservedQty: number;
  remainingShortQty: number;
  // Computed status badge:
  lineStatus: "AVAILABLE" | "RESERVED" | "SHORTAGE" | "PARTIAL" | "NO_DATA";
}
```

**Logic:**
```typescript
// 1) Verify template exists
const template = await getTemplateById(id);
if (!template) return NextResponse.json({ error: "NOT_FOUND" }, { status: 404 });

// 2) Load BOM tree (để có danh sách componentItemIds)
const tree = await loadTree(id);
const itemIds = tree.map(n => n.componentItemId);

// 3) Query bom_snapshot_line cho orderId, filter by component_item_id in itemIds
const snapLines = await db
  .select({
    componentItemId: bomSnapshotLine.componentItemId,
    componentSku: bomSnapshotLine.componentSku,
    grossRequiredQty: sql<number>`SUM(${bomSnapshotLine.grossRequiredQty})::float`,
    qcPassQty: sql<number>`SUM(${bomSnapshotLine.qcPassQty})::float`,
    reservedQty: sql<number>`SUM(${bomSnapshotLine.reservedQty})::float`,
    remainingShortQty: sql<number>`SUM(${bomSnapshotLine.remainingShortQty})::float`,
  })
  .from(bomSnapshotLine)
  .where(
    and(
      eq(bomSnapshotLine.orderId, orderId),
      inArray(bomSnapshotLine.componentItemId, itemIds),
    )
  )
  .groupBy(bomSnapshotLine.componentItemId, bomSnapshotLine.componentSku);

// 4) Build lookup map
const snapMap = new Map(snapLines.map(r => [r.componentItemId, r]));

// 5) Merge với BOM tree lines
const lines: OrderStatusLine[] = tree.map(node => {
  const snap = snapMap.get(node.componentItemId);
  if (!snap) return { componentItemId: node.componentItemId, componentSku: node.componentSku ?? "", ..., lineStatus: "NO_DATA" };
  const shortage = Number(snap.remainingShortQty ?? 0);
  const reserved = Number(snap.reservedQty ?? 0);
  const gross = Number(snap.grossRequiredQty ?? 0);
  let lineStatus: OrderStatusLine["lineStatus"];
  if (shortage > 0 && shortage >= gross) lineStatus = "SHORTAGE";
  else if (shortage > 0) lineStatus = "PARTIAL";
  else if (reserved > 0) lineStatus = "RESERVED";
  else lineStatus = "AVAILABLE";
  return { componentItemId: node.componentItemId, componentSku: node.componentSku ?? "", grossRequiredQty: gross, ..., lineStatus };
});

return NextResponse.json({ lines, orderInfo: { orderId, orderNo: order.orderNo } });
```

**Lấy `orderInfo`:** Query `sales_order` WHERE `id = orderId` để lấy `orderNo`, `customerName`, `status`.

export function useBomTemplateOrders(templateId: string | null) {
  return useQuery({
    queryKey: ["bom-template-orders", templateId],
    queryFn: async () => {
      if (!templateId) return { orders: [] };
      const res = await fetch(`/api/bom/templates/${templateId}/orders`);
      if (!res.ok) throw new Error("Lỗi tải danh sách đơn hàng");
      return res.json() as Promise<{ orders: OrderSummary[] }>;
    },
    enabled: !!templateId,
  });
}

export function useBomOrderStatus(templateId: string | null, orderId: string | null) {
  return useQuery({
    queryKey: ["bom-order-status", templateId, orderId],
    queryFn: async () => {
      if (!templateId || !orderId) return { lines: [] };
      const res = await fetch(`/api/bom/templates/${templateId}/order-status?orderId=${orderId}`);
      if (!res.ok) throw new Error("Lỗi tải trạng thái đơn hàng");
      return res.json() as Promise<{ lines: OrderStatusLine[], orderInfo: OrderInfo }>;
    },
    enabled: !!templateId && !!orderId,
  });
}

## Risks & Mitigations

| Risk | Mức độ | Mitigation |
|------|--------|-----------|
| Đổi accent màu gây visual regression | MEDIUM | Chỉ đổi token/variable, kiểm tra từng component chính. Giữ `blue-*` cho info/link semantic. |
| `bom_snapshot_line` chưa có data → Phase D dropdown trống | LOW | Seed demo không có `bom_revision` → không có snapshot. Phase D dropdown chỉ có data khi đã explode snapshot. Acceptable cho V1.5 demo. |
| `salesOrder.bomTemplateId` nullable → query orders of template có thể miss | MEDIUM | Seed demo phải set `bomTemplateId` đúng khi insert sales_order. |
| `date-fns` chưa có trong deps | LOW | Check `package.json` trước. Nếu chưa: `pnpm --filter @iot/web add date-fns`. |
| `BomTreeNodeRaw` type trong hook không update → TypeScript error | MEDIUM | Cần đọc `apps/web/src/hooks/useBom.ts` và update type trước khi sửa component. |
| Seed demo chạy 2 lần → duplicate key violation | LOW | Tất cả insert dùng `onConflictDoUpdate` hoặc `onConflictDoNothing`. Fixed UUIDs đảm bảo idempotent. |
| `supplier` table: kiểm tra columns trước khi seed | MEDIUM | Đọc schema/master.ts hoặc procurement.ts để biết `supplier` columns. Có thể cần: `code`, `name`, `contactEmail` — kiểm tra trước. |
