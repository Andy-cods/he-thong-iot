### Item 2 · BOM tree drag-drop cross-parent + auto level update
**Root cause / current state:**
- **Server READY:** `apps/web/src/server/repos/bomLines.ts` `moveLine()` ĐÃ có full logic cross-parent: `isDescendantOf` guard, `shiftSubtreeLevel` DFS recursive update level toàn subtree, depth check `currentMaxLevel + shift ≤ BOM_MAX_LEVEL=5`, transaction siblings position shift. Code xịn, đã test typecheck.
- **API READY:** `POST /api/bom/templates/[id]/lines/[lid]/move` forward input `newParentLineId + newPosition`.
- **Client BỎ KHÔNG DÙNG:** `BomTreeView.tsx` line 226-229 `handleDragEnd` — lấy `newParent = overNode.parentLineId` tức là luôn drop làm SIBLING của over node, không bao giờ drop INTO làm CHILD. Indent level hoàn toàn không thay đổi.
- **Thiếu UI affordance:** không có drop zone phân biệt "drop TRƯỚC over" vs "drop LÀM CON over" (pattern chuẩn: split row thành 3 zone trên-giữa-dưới, giữa = drop as child).

**Scope cần cook:**
- **2a** `BomTreeView.tsx` — dùng `useDroppable` riêng cho "drop as child" zone (middle 40% row height) + detect collision zone trong `handleDragEnd`:
  - Zone TOP 30% → drop làm sibling TRƯỚC over (newParent = over.parentLineId, newPosition = over.position)
  - Zone MIDDLE 40% → drop làm CHILD của over (newParent = over.id, newPosition = 1)
  - Zone BOTTOM 30% → drop làm sibling SAU over (newParent = over.parentLineId, newPosition = over.position + 1)
  - Dùng `pointerWithin` + manual event.delta.y calculation.
- **2b** Visual feedback: khi drop zone MIDDLE → hover row highlight border-l-4 blue-500 + indent line dash. Hiện chỉ có `isDragging` shadow-md.
- **2c** Level badge client optimistic update: sau `onMove` TanStack Query invalidate tree → refetch `/api/bom/templates/[id]/tree` → re-render với level mới. Đã có optimistic logic trong `useBom.ts`, chỉ cần cập nhật level field trong cache.
- **2d** Keyboard cross-parent: `Shift+ArrowRight` = indent (thành child của sibling trước), `Shift+ArrowLeft` = outdent (thành sibling của parent). Outlook-style. (nice-to-have)

**File + API + endpoint:**
- `apps/web/src/components/bom/BomTreeView.tsx` (major refactor `handleDragEnd` + add zone detection + `useDroppable`)
- `apps/web/src/hooks/useBom.ts` (verify optimistic update cho level field)
- KHÔNG đụng server — sẵn rồi

**Effort:** 1-1.5 ngày
**Priority:** **P0** — UX lớn, code nhỏ, server đã có sẵn, bỏ qua = phí
**Dependency:** không block. Parallel được.
**Acceptance:**
1. Kéo L2 component từ parent A → drop MIDDLE parent B (khác cây) → server nhận `newParentLineId = parentB.id`, tree refetch hiển thị component ở đúng vị trí L3 dưới B
2. Kéo L1 → drop MIDDLE L1 khác → tạo quan hệ L1-parent/L2-child, level badge tự update 2→2
3. Kéo → drop vào descendant của chính nó → client block (hiện ancestorIds đã check), hoặc server trả 400 `CANNOT_MOVE_INTO_DESCENDANT`
4. Kéo subtree depth 3 → drop vào L2 (sẽ thành L3 → L4 → L5) → ok. Drop vào L3 → server 400 `MAX_DEPTH_EXCEEDED`, toast error
5. Keyboard `Shift+ArrowRight` indent: chọn row → key → thành child của sibling trước (V1.1-beta optional)

### 3.2 Tree cross-parent drag
- **Zone detection:** manual compute `event.delta.y` offset vs `over.rect.top` + row height 32px → 3 zone 30/40/30%.
- **@dnd-kit approach:** `useDroppable` trong mỗi TreeRow với id suffix `:child` cho zone middle. `handleDragEnd` check `over.id.endsWith(':child')` → set `newParent = over.id.split(':')[0]`.
- **Client-side guard redundant với server:** check `activeNode.ancestorIds.includes(overNode.id)` + `max depth` trước khi gọi API → toast error ngay không round-trip.
- **Optimistic update:** `useMoveBomLine` mutation trong `useBom.ts` — update cache với `parentLineId + position + level` mới NGAY. Rollback on error. Level recalc descendants: DỄ sai — làm KHÔNG optimistic level, chờ refetch. Position + parent thì optimistic.

- **Pattern 1 `zod default → strict`:** dùng `z.input<typeof schema>` vs `z.output<typeof schema>` hoặc sửa repo signature nhận `Partial<X>` + merge default server-side.

Verify `packages/shared/src/schemas/supplier.ts` có `.default()` chưa — nếu có thì là bug Drizzle version, pin zod version.
- **Pattern 2 `row possibly undefined`:** guard 1 dòng `if (!row) throw notFound()`.
- **Pattern 3 `AuditAction enum thiếu`:** thêm `UPLOAD | COMMIT` vào `packages/shared/src/types/audit.ts` type union + nếu có DB enum (CHECK constraint) → migration mini 0004 `ALTER TYPE app.audit_action ADD VALUE`.
- **Pattern 4 `"true"|"false"` query string:** `z.coerce.boolean()` hoặc `.transform((v) => ["true","1","yes"].includes(v))`.

## §5 Risk register (5 risks + mitigation)

### R1 · Worker fail khi import 4 sheet đồng thời (race condition auto-create item)
- **Likelihood:** HIGH — file sample có sheet 238846/238847/238848/238851 cùng đụng SKU chung (BOM variant cùng base)
- **Impact:** MEDIUM — 1 chunk crash → rollback TX, mất rows chunk đó, ghi error
- **Mitigation:**
  - Fix `ON CONFLICT DO UPDATE` trên `item.sku` (decision 3.4)
  - Worker queue concurrency = 1 (BullMQ default) → xử lý sequential theo job, không race cross-job
  - Trong 1 job, sheet xử lý sequential trong outer loop → không race intra-job
  - Test case: upload file sample, verify log worker không có `unique_violation`

### R2 · Tree cross-parent level recalc cyclic ref
- **Likelihood:** LOW — `moveLine` đã có `isDescendantOf` guard
- **Impact:** HIGH — nếu miss, DB bị cycle, tree infinite loop render → crash client
- **Mitigation:**
  - Server guard đã có
  - Client duplicate guard `ancestorIds.includes(over.id)` trước khi gọi API
  - Test case: drag L1 → drop vào child của chính nó → expect toast error, không round-trip
  - Test case: 3-level deep subtree, drop ancestor vào descendant → server trả 400

### R3 · Dashboard aggregate query slow (no index, full scan)
- **Likelihood:** LOW — 3 bảng nhỏ (item < 1000, bom_template < 100, supplier < 50)
- **Impact:** LOW — first request < 300ms, cached request < 50ms
- **Mitigation:**
  - Redis cache 60s
  - `EXPLAIN ANALYZE` count query với 10k item → phải < 50ms (Postgres count(*) với index on is_active)
  - Index đã có `item_is_active_idx` V1 foundation (verify `packages/db/migrations/0001_init.sql`)
  - Monitor: cache miss ratio log → nếu > 30% thì tăng TTL 120s

### R4 · TS fix phá runtime (zod default mismatch → 400 nơi đang 200)
- **Likelihood:** MEDIUM — `z.coerce.boolean()` thay behavior string → boolean
- **Impact:** MEDIUM — nếu FE gửi "true" string, cũ: reject, mới: accept. Thực tế FE gửi gì?
- **Mitigation:**
  - Audit FE usage: grep `useItems`, `useSuppliers` trong `apps/web/src/hooks` xem query params shape
  - Test curl từng endpoint với payload cũ (`?isActive=true` string) sau fix → phải 200
  - Test với payload mới (`?isActive=1`, `?isActive=yes`) → 200
  - Refactor tiny step — fix 1 file, `pnpm test`, next

### R5 · Mobile responsive breakpoint collision với desktop
- **Likelihood:** MEDIUM — `useMediaQuery` SSR mismatch phổ biến Next.js
- **Impact:** LOW — hydration warning, visual flicker 100ms
- **Mitigation:**
  - Hook `useMediaQuery` dùng `useSyncExternalStore` (React 18 safe) + initial `false` SSR, cập nhật sau mount
  - Render mobile-first Tailwind classes → desktop override. Không JS-based conditional render khi có CSS solution.
  - Test suite: Chrome DevTools device mode iPhone 12 + iPad Mini screenshot before/after
  - `lighthouse --preset=desktop + mobile` compare score
