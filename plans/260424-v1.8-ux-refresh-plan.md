### Các phát hiện quan trọng

1. **BarcodeScanner đã là component standalone** → Phase D reuse gần như zero-effort.
2. **Receiving backend tồn tại, chỉ thiếu lớp "chọn PO"** → cần API `GET /api/purchase-orders?status=OPEN` + thay `DEMO_POS` static bằng fetch thật. Stub `/api/po/[id]` còn trả demo → cần mở rộng để đọc `purchase_order` + `purchase_order_line` thật.
3. **"Dòng sản phẩm" ≠ "Danh mục vật tư"**. Dòng sản phẩm = nhóm mã Z (Z0000002…). Danh mục = SKU master. User yêu cầu "gộp" có thể hiểu:
   - (a) Bỏ 1 trong 2 khỏi sidebar → chỉ giữ link sâu khi cần. **Không khả thi** vì 2 dataset khác nhau.
   - (b) Đổi nhóm sidebar: gom "Dòng SP" + "BOM Templates" thành nhóm "Thiết kế / Công thức" (dưới 1 divider). **Khả thi, safe.**
   - (c) Thay "Dòng sản phẩm" top-level bằng tab trong /bom hub, user truy cập qua /bom?tab=product-lines. **Làm việc thêm, breaking link.** Không chọn.
   - → **Chọn (b)**. Giữ cả 2 route, tổ chức lại visual group trong sidebar.
4. **Items ↔ BOM** link 1 chiều đã có (BomLineInspector click → /items/[id]). Chiều ngược thiếu.

**Logic match SKU → line:**
```ts
function findLinesByCode(tree, code) {
  // match theo:
  //   1. componentItem.sku === code (exact)
  //   2. OR code là một barcode có trong bảng item_barcode (cần API thêm)
  //      fallback V1.8: chỉ SKU exact match; barcode lookup V1.9.
  return tree.flatMap(row => row.componentItem.sku === code ? [row] : []);
}
```
- Nếu match 1 line → scroll vào view + flash border-green 600ms + toast "Đã tìm thấy {sku} (dòng {lineNo})".
- Nếu match nhiều line (cùng SKU ở nhiều level) → highlight tất cả + toast "Tìm thấy X dòng".
- Nếu không match → toast warning "SKU {code} không có trong BOM này" + beep error từ BarcodeScanner.
- Dup scan trong 800ms → BarcodeScanner auto beep dup (logic built-in).

**Acceptance:**
- [ ] Camera + USB wedge + manual đều fire callback
- [ ] Match SKU → row tương ứng flash viền xanh + scroll smoothly
- [ ] Dialog đóng ESC hoặc backdrop
- [ ] Sound toggle persist trong session (localStorage)
- [ ] Không ảnh hưởng hotkeys hiện tại của BomGridPro

**Rủi ro:**
- `html5-qrcode` bundle size ~200KB gzip → lazy import đã có sẵn, không preload.
- Camera permission HTTPS-only → production OK (mes.songchau.vn có TLS), dev `localhost` OK.

  ```sql
  SELECT bt.id, bt.code, bt.name, bt.status, bl.line_no, bl.quantity_per, bl.unit_of_measure, bl.parent_line_id
  ```

## Suggested rollout order

Thứ tự tối ưu theo **rủi ro tăng dần + value ship được từng chunk**:
