## 1. Goal + Success criteria

### Goal (user story)
> "Tôi mở BOM → thấy bảng grid chuyên nghiệp 13 cột giống Excel nhưng có cột **Tiến độ** (progress bar + status badge) và cột **Actions** sticky bên phải. Hover vào 1 dòng thấy 4 nút (Đặt mua / Xem tồn / Sửa / More). Click Sửa mở side sheet 480px — fix số lượng, hao hụt, ghi chú xong Save là refresh. Click Đặt mua mở prefill PR form. Đủ nhanh, nhìn ngon, không nặng như Univer."

### Success criteria (measurable)

1. **Visual**
   - 13 cột render đúng thứ tự: `# | Ảnh | Mã | Tên | Loại | Vật liệu | NCC | SL/bộ | Kích thước | Tổng SL | Hao hụt % | Tiến độ | Ghi chú | [Actions sticky]`.
   - Header row sticky top + 2 cột trái sticky left + 1 cột phải sticky right.
   - Progress cell render đủ 5 states (PLANNED/PURCHASING/PARTIAL/AVAILABLE/ISSUED) với bar + badge distinct color.
   - Action icons ẩn default, hover row → visible (opacity 0 → 1, 150ms).
2. **Bundle**
   - `/bom/[id]/grid` bundle gzip giảm từ ~1.2MB → ~400KB (drop `@univerjs/*` ~800KB, add `@tanstack/react-table` ~14KB gzip).
3. **Smoke tests** — `tests/smoke/v1.7-smoke.sh` giữ 43/43 cũ + thêm 12-15 assertion mới:
   - `/bom/[id]/grid` 200 + HTML chứa `data-testid="bom-grid-pro"` (phase A).
   - HTML chứa 13 `<th>` với aria-label đúng tên cột.
   - Endpoint `/api/bom/templates/[id]/lines-with-status` 200 + JSON shape `lines[].derivedStatus` exist (phase B).
   - Endpoint `PATCH /api/bom/templates/[id]/lines/[lineId]` đã tồn tại sẵn — giữ 200 với body `{ qtyPerParent: "5" }`.
   - `?mode=univer` fallback render Univer HTML có marker `iot_univer` (phase D).
   - `NEXT_PUBLIC_BOM_GRID_PRO=1` default bật → route mặc định là pro.
4. **Manual E2E** (login admin, Firefox + Chrome):
   - Hover row → thấy 4 icon action (Đặt mua / Xem tồn / Sửa / ⋯).
   - Click Sửa → sheet 480px mở, form chứa qtyPerParent + scrapPercent + description + supplierItemCode.
   - Edit qtyPerParent 2 → 5 → Save → toast success, row re-render qty=5, DB `bom_line.qty_per_parent = '5.000'`.
   - Click Đặt mua (status PLANNED) → `PRQuickDialog` mở prefill item + qty=totalShort.
   - Expand/collapse group row (depth 1) → children hidden/visible với animation.
5. **Typecheck + build** mỗi commit: `pnpm --filter @iot/web typecheck` 0 new error, `pnpm --filter @iot/web build` PASS.

### C. flatten-tree.ts (commit A1)

```ts
import type { BomTreeNodeRaw } from "@/hooks/useBom";

export interface FlatBomRow extends BomTreeNodeRaw {
  isGroup: boolean;
  depth: number;
  rowIndex: number;
  children?: FlatBomRow[]; // cho getSubRows Tanstack
}

export function flattenBomTreeToRows(tree: BomTreeNodeRaw[]): FlatBomRow[] {
  const childrenMap = new Map<string | null, BomTreeNodeRaw[]>();
  for (const n of tree) {
    const key = n.parentLineId;
    if (!childrenMap.has(key)) childrenMap.set(key, []);
    childrenMap.get(key)!.push(n);
  }
  childrenMap.forEach(arr => arr.sort((a, b) => a.position - b.position));

  let idx = 0;
  const build = (parentId: string | null, depth: number): FlatBomRow[] => {
    return (childrenMap.get(parentId) ?? []).map(n => {
      const isGroup = n.childCount > 0;
      const row: FlatBomRow = { ...n, isGroup, depth, rowIndex: idx++ };
      if (isGroup) row.children = build(n.id, depth + 1);
      return row;
    });
  };
  return build(null, 0);
}
```
