# V1.7-beta.2 Phase C — Manual Test Plan

**Ngày:** 2026-04-21
**Tác giả:** cook agent
**Scope:** Wire 4 action button trong `BomGridPro` từ toast placeholder → thực.
**Environment:** LIVE `https://mes.songchau.vn` (admin / ChangeMe!234)

---

## 0. Chuẩn bị

1. Đăng nhập admin tại `https://mes.songchau.vn/login`.
2. Mở 1 BOM bất kỳ có linh kiện, VD `/bom/{id}/grid` (pick qua `/bom` list).
3. Mở DevTools Console (F12) — expect không có lỗi đỏ khi render grid.

---

## 1. Nút **Sửa (Pencil)** → `BomLineSheet`

**Steps:**
1. Hover 1 dòng linh kiện bất kỳ (non-group, non-OBSOLETE).
2. Click icon pencil.

**Expected:**
- Side Sheet slide-in từ phải, rộng ~480px.
- Header: "Chỉnh sửa linh kiện" + BOM code + SKU badge.
- Form có 6 field: SKU readonly, Tên readonly, Số lượng/bộ, Hao hụt %, ĐVT, Kích thước, Mã NCC, Mô tả.
- Fields prefill đúng giá trị hiện tại.

**Test submit:**
3. Sửa **Số lượng/bộ** từ 2 → 5, **Hao hụt %** từ 0 → 3.
4. Click **Lưu**.

**Expected:**
- Toast success: "Đã lưu thay đổi".
- Sheet tự đóng.
- Row trong grid cập nhật qty = 5 (tự invalidate cache).
- DevTools Network: thấy `PATCH /api/bom/templates/{id}/lines/{lid}` → 200.

**Validation edge:**
5. Reopen Sheet, clear Số lượng → Lưu → expect error inline "Số lượng phải ≥ 0.01".
6. Nhập Hao hụt = 150 → expect error "Hao hụt 0-100%".

---

## 2. Nút **Đặt mua (ShoppingCart)** → `PRQuickDialog`

**Steps:**
1. Hover 1 dòng linh kiện, click icon giỏ hàng.

**Expected:**
- Dialog center ~480px "Đặt mua nhanh — {SKU}".
- Description mô tả PR DRAFT.
- Item card hiển thị SKU + tên.
- Số lượng đề xuất prefill = ceil(qtyPerParent × parentQty × 1.1, 3 decimals).
- Priority = NORMAL.
- Ghi chú = `Từ BOM {code} — dòng {sku}`.
- Checkbox "Mở PR sau khi tạo" checked.

**Test submit:**
2. Click **Tạo PR**.

**Expected:**
- Toast success "Đã tạo PR {code}".
- Nếu checkbox checked → redirect `/procurement/purchase-requests/{id}`.
- PR mới xuất hiện trong list DRAFT với 1 dòng đúng item + qty.
- DevTools Network: `POST /api/purchase-requests` → 201.

**Cancel flow:**
3. Mở lại dialog trên dòng khác → click **Huỷ** → dialog đóng, không tạo PR.

---

## 3. Nút **Xem tồn (Package)** → `InventoryPopover`

**Steps:**
1. Hover 1 dòng linh kiện, click icon Package.

**Expected:**
- Popover 360px align-right mở ra cạnh icon.
- Header: tên + SKU component.
- Loading spinner ngắn.
- 4 KPI cell: Tổng tồn · Sẵn dùng · Giữ QC · Đã giữ chỗ (số formatted `1.234`).
- Section "Lot gần nhất" list tối đa 5 lot với: mã lot/serial, badge status (Sẵn dùng/Giữ QC/Đã dùng/Hết hạn), HSD nếu có, qty on-hand.
- Footer link "Xem tất cả lot" → `/lot-serial?itemId={id}`.

**Edge case:**
2. Click icon Package cho item **chưa có lot** → popover hiển thị "Chưa có lot nào cho linh kiện này." + 4 KPI đều = 0.
3. Click Package lần 2 trong 30s → không re-fetch (staleTime 30s).

**Network check:**
- DevTools: `GET /api/items/{itemId}/inventory-summary` → 200, response JSON có `data.summary` + `data.lots`.

## 4. Dropdown **More (...)** → Nhân bản + Lịch sử + Xoá

**Steps:**

**Expected menu items:**
- Lịch sử
- Nhân bản dòng

- Xoá (đỏ)

**Test Nhân bản:**

**Expected:**

- Row mới xuất hiện ngay dưới row gốc cùng SKU (position + 1).
- DB: thêm 1 `bom_line` mới cùng `componentItemId`.

**Test Lịch sử:**
3. Click "Lịch sử".

**Expected:**
- HistoryDrawer slide-in từ phải (drawer BOM workspace có sẵn).

**Test Xoá:**
4. Click "Xoá" (red) → DialogConfirm xuất hiện.
5. Gõ `XOA` → click "Xoá".

**Expected:**
- Toast success "Đã xoá {SKU}.".
- Row biến mất khỏi grid.
- Nếu row có children → description đề cập "cascade N linh kiện con".
