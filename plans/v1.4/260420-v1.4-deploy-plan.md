## Phase 1 — Prune VPS để giải phóng disk (10 phút)

**Mục tiêu:** giải phóng >= 5.5GB để đạt total free >= 14GB.
**Chạy trên VPS:**

```bash
# 1a. Xóa image cũ SAFE (chỉ xóa tag rác — giữ v1.3 và local)
# Lấy Image ID của v1.3 trước để không xóa nhầm
V13_ID=$(docker inspect --format='{{.Id}}' hethong-iot:v1.3 2>/dev/null | head -c 12)
echo "v1.3 ID prefix: $V13_ID"

# Xóa các tag cũ theo tên (không xóa nếu cùng ID với v1.3/local)
for tag in v1.3-prev v1.1-alpha v1.1-alpha-prev v1.1-alpha.1 v1.1-beta v1.2 v2; do
  IMG_ID=$(docker inspect --format='{{.Id}}' "hethong-iot:${tag}" 2>/dev/null | head -c 12)
  if [ -n "$IMG_ID" ] && [ "$IMG_ID" != "$V13_ID" ]; then
    echo "Removing hethong-iot:${tag} (ID: $IMG_ID)"
    docker rmi "hethong-iot:${tag}" || true
  else
    echo "Skipping hethong-iot:${tag} (not found or same as v1.3)"
  fi
done

# 1b. Prune builder cache (SAFE — không ảnh hưởng running containers)
docker builder prune -f
# Kỳ vọng: giải phóng 4-6GB BuildKit cache

# 1c. Prune dangling images + stopped containers + unused networks
# (KHÔNG dùng -a vì sẽ xóa image không dùng bao gồm cả image base)
docker system prune -f
# Kỳ vọng: dangling images + build cache nhỏ còn lại

# 1d. Dọn apt cache và journal
apt-get clean
journalctl --vacuum-time=7d
# Journal chỉ giữ 7 ngày gần nhất

# 1e. Kiểm disk sau prune — PHẢI >= 14GB free trước khi tiếp tục
df -h /
docker system df
```

**Checkpoint Phase 1:**
- `df -h /` phải thấy Available >= 14G
- Nếu chưa đủ: `docker image prune -a --filter "until=720h" -f` (xóa image cũ > 30 ngày, nhưng KIỂM tra không ảnh hưởng v1.3)
- Nếu vẫn chưa đủ: escalate, kiểm `/var/lib/docker/overlay2` size

**Nếu build fail:**
- `ERROR: failed to solve: process "/bin/sh -c pnpm build"` → kiểm pnpm-lock.yaml đồng bộ: `pnpm install --frozen-lockfile` trước
- `COPY failed: file not found` → kiểm `.dockerignore` không loại trừ nhầm file cần thiết
- OOM kill → tăng Docker Desktop memory limit lên >= 4GB trong Settings → Resources

## Phase 5 — Migration 0007a + 0007b (10 phút)

**QUAN TRỌNG:** `0007a` PHẢI chạy bằng `postgres` user (superuser). `0007b` chạy bằng `hethong_app`.

**Chạy trên VPS:**

```bash
# 5a. Verify postgres container healthy
docker exec iot_postgres pg_isready -U hethong_app -d hethong_iot
# Kỳ vọng: "accepting connections"

# 5b. Kiểm migration 0007a/0007b chưa apply (optional, idempotent nhưng cần chắc)
docker exec -i iot_postgres psql -U hethong_app -d hethong_iot -c \
  "SELECT rolname FROM pg_roles WHERE rolname LIKE 'iot_%';"
# Nếu đã có iot_admin/iot_planner/... → 0007a đã apply, vẫn OK chạy lại (idempotent)

docker exec -i iot_postgres psql -U hethong_app -d hethong_iot -c \
  "SELECT column_name FROM information_schema.columns WHERE table_name='user_account' AND column_name='must_change_password';"
# Nếu trả về row → 0007b đã apply

# ------------------------------------------------------------------
# 5c. Apply migration 0007a — PHẢI dùng user postgres (superuser)
# ------------------------------------------------------------------
MIGRATIONS_SRC="/tmp/iot-migrations"
# Nếu dùng repo path: MIGRATIONS_SRC="/opt/hethong-iot/repo/packages/db/migrations"

docker cp "${MIGRATIONS_SRC}/0007a_rls_policies.sql" iot_postgres:/tmp/0007a_rls_policies.sql

docker exec -i iot_postgres psql \
  -U postgres \
  -d hethong_iot \
  -v ON_ERROR_STOP=1 \
  -f /tmp/0007a_rls_policies.sql

# Kỳ vọng cuối: NOTICE: migration 0007a: RLS policies applied — 10 tables × avg 3 policy = ~33 policy.

# ------------------------------------------------------------------
# 5d. Apply migration 0007b — dùng hethong_app (không cần superuser)
# ------------------------------------------------------------------
docker cp "${MIGRATIONS_SRC}/0007b_session_enhance.sql" iot_postgres:/tmp/0007b_session_enhance.sql

docker exec -i iot_postgres psql \
  -U hethong_app \
  -d hethong_iot \
  -v ON_ERROR_STOP=1 \
  -f /tmp/0007b_session_enhance.sql

# Kỳ vọng: COMMIT (không error)

# 5e. Verify migration 0007a — roles tồn tại
docker exec -i iot_postgres psql -U hethong_app -d hethong_iot -c \
  "SELECT rolname FROM pg_roles WHERE rolname LIKE 'iot_%' ORDER BY rolname;"
# Kỳ vọng: iot_admin, iot_operator, iot_planner, iot_warehouse

# 5f. Verify migration 0007b — columns mới tồn tại
docker exec -i iot_postgres psql -U hethong_app -d hethong_iot -c \
  "SELECT column_name, data_type FROM information_schema.columns
   WHERE table_schema='app' AND table_name='user_account'
   AND column_name IN ('must_change_password','password_reset_token_hash','password_reset_expires_at')
   ORDER BY column_name;"
# Kỳ vọng: 3 rows

docker exec -i iot_postgres psql -U hethong_app -d hethong_iot -c \
  "SELECT column_name, data_type FROM information_schema.columns
   WHERE table_schema='app' AND table_name='session'
   AND column_name = 'last_seen_at';"
# Kỳ vọng: 1 row

# 5g. Verify RLS enabled trên 10 bảng
docker exec -i iot_postgres psql -U hethong_app -d hethong_iot -c \
  "SELECT tablename, rowsecurity FROM pg_tables
   WHERE schemaname='app' AND rowsecurity=true ORDER BY tablename;"
# Kỳ vọng: 10 bảng: audit_event, bom_revision, bom_snapshot_line, bom_template,
#           eco_change, item, reservation, sales_order, supplier, work_order

# 5h. Dọn migration files tmp
docker exec iot_postgres rm -f /tmp/0007a_rls_policies.sql /tmp/0007b_session_enhance.sql
```

**Xử lý lỗi migration:**

| Lỗi | Nguyên nhân | Fix |
|---|---|---|
| `ERROR: permission denied to create role` | Chạy nhầm user `hethong_app` cho 0007a | Dùng `-U postgres` |
| `ERROR: column "must_change_password" of relation "user_account" already exists` | 0007b đã apply lần trước | OK, migration dùng `IF NOT EXISTS`, bỏ qua |
| `ERROR: role "iot_admin" already exists` | 0007a đã apply lần trước | OK, DO block dùng `IF NOT EXISTS`, bỏ qua |
| `ERROR: FATAL: role "postgres" does not exist` | Container postgres chạy với user khác | Kiểm: `docker exec iot_postgres psql -U postgres -c '\du'` |

**Chạy trên VPS:**


## Phase 8 — Smoke test (10 phút)

**Chạy LOCAL Windows bash hoặc VPS (cả hai đều được):**

Thay `COOKIE` bằng giá trị nhận từ Set-Cookie của bước 3.

```bash
BASE="https://mes.songchau.vn"
# Hoặc test qua IP trực tiếp (bỏ qua Caddy):
# BASE_INTERNAL="http://127.0.0.1:3001"  # từ VPS

# ----------------------------------------------------------------
# Case 1: GET /api/health → 200
# ----------------------------------------------------------------
echo "=== Case 1: /api/health ==="
curl -sf "${BASE}/api/health" && echo " PASS" || echo " FAIL"

# ----------------------------------------------------------------
# Case 2: GET /api/ready → 200
# ----------------------------------------------------------------
echo "=== Case 2: /api/ready ==="
curl -sf "${BASE}/api/ready" && echo " PASS" || echo " FAIL"

# ----------------------------------------------------------------
# Case 3: POST /api/auth/login (đúng) → 200 + Set-Cookie
# ----------------------------------------------------------------
echo "=== Case 3: Login đúng ==="
LOGIN_RESP=$(curl -sf -c /tmp/iot-cookies.txt \
  -X POST "${BASE}/api/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"ChangeMe!234"}')
echo "$LOGIN_RESP" | head -c 200
echo ""
grep -q 'iot_session\|iot-session\|auth-token' /tmp/iot-cookies.txt && echo " Cookie: PASS" || echo " Cookie: FAIL"

# ----------------------------------------------------------------
# Case 4: GET /api/auth/me với cookie → 200 + role admin
# ----------------------------------------------------------------
echo "=== Case 4: /api/auth/me ==="
ME_RESP=$(curl -sf -b /tmp/iot-cookies.txt "${BASE}/api/auth/me")
echo "$ME_RESP" | head -c 200
echo ""
echo "$ME_RESP" | grep -q '"admin"' && echo " Role admin: PASS" || echo " Role admin: FAIL"

# ----------------------------------------------------------------
# Case 5: GET /api/items → 200 + array
# ----------------------------------------------------------------
echo "=== Case 5: /api/items ==="
ITEMS_RESP=$(curl -sf -b /tmp/iot-cookies.txt "${BASE}/api/items?limit=5")
echo "$ITEMS_RESP" | head -c 200
echo ""
echo "$ITEMS_RESP" | grep -qE '"data"\s*:\s*\[' && echo " PASS" || echo " FAIL"

# ----------------------------------------------------------------
# Case 6: GET /api/bom/templates → 200 + array
# ----------------------------------------------------------------
echo "=== Case 6: /api/bom/templates ==="
BOM_RESP=$(curl -sf -b /tmp/iot-cookies.txt "${BASE}/api/bom/templates")
echo "$BOM_RESP" | head -c 200
echo ""
echo "$BOM_RESP" | grep -qE '"data"\s*:\s*\[' && echo " PASS" || echo " FAIL"

# ----------------------------------------------------------------
# Case 7: POST /api/auth/login (sai pass) → 401 (KHÔNG 500)
# ----------------------------------------------------------------
echo "=== Case 7: Login sai pass (kỳ vọng 401) ==="
HTTP_CODE=$(curl -s -o /dev/null -w "%{http_code}" \
  -X POST "${BASE}/api/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"WrongPassword!"}')
[ "$HTTP_CODE" = "401" ] && echo " 401: PASS" || echo " Got $HTTP_CODE: FAIL"

# ----------------------------------------------------------------
# Case 8: Rate limit — 5 request sai liên tiếp từ cùng IP → 429
# (V1.4 feature: 5 lần sai trong 60s → rate limit)
# ----------------------------------------------------------------
echo "=== Case 8: Rate limit login (5 lần sai → 429) ==="
for i in 1 2 3 4 5 6; do
  CODE=$(curl -s -o /dev/null -w "%{http_code}" \
    -X POST "${BASE}/api/auth/login" \
    -H "Content-Type: application/json" \
    -d "{\"username\":\"admin\",\"password\":\"WrongPass${i}!\"}")
  echo "  Attempt $i: HTTP $CODE"
done
# Kỳ vọng: lần 1-5 → 401, lần 6 → 429
# Nếu tất cả đều 401: kiểm tra Redis connection + RateLimit middleware mount
# Xử lý: xem Phase 8 troubleshooting bên dưới

# ----------------------------------------------------------------
# Case 9: GET /api/work-orders → 200
# ----------------------------------------------------------------
echo "=== Case 9: /api/work-orders ==="
WO_RESP=$(curl -sf -b /tmp/iot-cookies.txt "${BASE}/api/work-orders")
echo "$WO_RESP" | head -c 200
echo ""
echo "$WO_RESP" | grep -qE '"data"\s*:\s*\[' && echo " PASS" || echo " FAIL"

# ----------------------------------------------------------------
# Case 10: GET /api/admin/audit → 200
# ----------------------------------------------------------------
echo "=== Case 10: /api/admin/audit ==="
AUDIT_RESP=$(curl -sf -b /tmp/iot-cookies.txt "${BASE}/api/admin/audit")
echo "$AUDIT_RESP" | head -c 200
echo ""
echo "$AUDIT_RESP" | grep -qE '"data"\s*:\s*\[' && echo " PASS" || echo " FAIL"

# ----------------------------------------------------------------
# Case 11: GET /api/admin/sessions/me → 200 (V1.4 session mgmt)
# ----------------------------------------------------------------
echo "=== Case 11: /api/admin/sessions/me ==="
SESS_RESP=$(curl -sf -b /tmp/iot-cookies.txt "${BASE}/api/admin/sessions/me")
echo "$SESS_RESP" | head -c 200
echo ""
# Kỳ vọng: trả về array sessions với current session
echo "$SESS_RESP" | grep -qE '\[' && echo " PASS" || echo " FAIL"

# ----------------------------------------------------------------
# Case 12: HTTPS check via Caddy → 200
# ----------------------------------------------------------------
echo "=== Case 12: HTTPS via Caddy ==="
HTTPS_CODE=$(curl -sI "https://mes.songchau.vn/api/health" -o /dev/null -w "%{http_code}")
[ "$HTTPS_CODE" = "200" ] && echo " HTTPS 200: PASS" || echo " Got $HTTPS_CODE: FAIL"

# Dọn cookies
rm -f /tmp/iot-cookies.txt
```

### 10b. Tạo script `deploy/scripts/docker-cleanup.sh` (manual trigger)

**Chạy LOCAL:**

```bash
cat > /c/dev/he-thong-iot/deploy/scripts/docker-cleanup.sh <<'EOF'
#!/usr/bin/env bash
# =============================================================
# docker-cleanup.sh — Manual Docker maintenance
# V1.4 Phase 8 — Cleanup rule
#
# Dùng khi: disk usage > 70%, trước build mới, sau rollback.
# KHÔNG xóa: volume iot_pg_data, iot_redis_data, iot_caddy_data
# KHÔNG xóa: image đang running (docker prune tự skip)
# =============================================================
set -euo pipefail

LOG_FILE="${LOG_FILE:-/var/log/docker-prune.log}"
echo "[$(date '+%Y-%m-%d %H:%M:%S')] === docker-cleanup.sh start ===" | tee -a "$LOG_FILE"

# 1. Disk trước cleanup
echo "[disk-before]" | tee -a "$LOG_FILE"
df -h / | tee -a "$LOG_FILE"
docker system df | tee -a "$LOG_FILE"

# 2. BuildKit cache (safe — không ảnh hưởng container)
echo "[builder-prune]" | tee -a "$LOG_FILE"
docker builder prune -f 2>&1 | tee -a "$LOG_FILE"

# 3. Image cũ hơn 30 ngày (image đang dùng bởi container sẽ bị skip tự động)
echo "[image-prune-30d]" | tee -a "$LOG_FILE"
docker image prune -a --filter "until=720h" -f 2>&1 | tee -a "$LOG_FILE"

# 4. Dangling objects (stopped containers, unused networks, dangling volumes)
# KHÔNG dùng --volumes vì sẽ xóa iot_pg_data!
echo "[system-prune-no-volumes]" | tee -a "$LOG_FILE"
docker system prune -f 2>&1 | tee -a "$LOG_FILE"

# 5. Disk sau cleanup
echo "[disk-after]" | tee -a "$LOG_FILE"
df -h / | tee -a "$LOG_FILE"
docker system df | tee -a "$LOG_FILE"

echo "[$(date '+%Y-%m-%d %H:%M:%S')] === docker-cleanup.sh done ===" | tee -a "$LOG_FILE"
EOF

chmod +x /c/dev/he-thong-iot/deploy/scripts/docker-cleanup.sh
```
