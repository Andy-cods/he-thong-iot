## Research findings — Barcode HID 2025/2026

**Kết luận ngắn:** USB/Bluetooth barcode scanner phổ thông đều mặc định HID keyboard-wedge mode (Plug & Play trên Win/Android/iOS). Trình duyệt không phân biệt scanner và bàn phím → phải dùng **timing detection**: scanner gõ ký tự liên tiếp < 50ms, bàn phím người gõ > 100–300ms.

**Thư viện được khuyến nghị (đánh giá 2025):**

| Library | Approach | Khuyến nghị |
|---|---|---|
| `onscan.js` (axenox) | Vanilla JS, timing + prefix/suffix detect, clipboard fallback | ✅ Production-ready, 6k★, SSR-safe |
| `use-scan-detection` (markjaniczak) | React hook wrapper, TypeScript native | ✅ Tốt nhất cho React/Next.js |
| `@zxing/library` + `html5-qrcode` | Camera-based (WebRTC + Barcode Detection API) | ✅ Đã có trong repo — dùng cho tablet/mobile |
| `WebHidBarcodeScanner` (NielsLeenheer) | WebHID low-level access | ⚠️ Chỉ Chromium desktop, yêu cầu user grant permission per device |

**Best practice áp dụng:**
1. **Primary:** HID keyboard-wedge detection với `use-scan-detection` — không cần user bấm gì, scanner gõ → auto-capture.
2. **Threshold:** Giữa các ký tự < 50ms + tổng > 4 ký tự + kết thúc bằng Enter (CR).
3. **UX:** Phát beep + vibration (`navigator.vibrate`) khi detect thành công; visual flash toast.
4. **Fallback camera:** Giữ `html5-qrcode` hiện có cho máy không gắn scanner.
5. **Cấu hình scanner:** Doc cho admin — set scanner suffix = Enter (\r) để chắc ăn; hoặc prefix ~ (tilde) để phân biệt với typing nhanh.

**Ảnh hưởng tới code:**
- Thêm dependency: `use-scan-detection` (~2KB) hoặc tự viết hook đơn giản (~50 LOC).
- Listener gắn ở level `RootLayout` (global) hoặc per-page (`/assembly`, `/receiving`, `/work-orders/[id]`).
- Dispatch custom event `barcode:scanned` → các page lắng nghe riêng.

**Recommendation:** Viết hook `useBarcodeScan` tự host (tránh dep ngoài) dựa trên logic `use-scan-detection`:
```ts
// apps/web/src/hooks/useBarcodeScan.ts (pseudocode ~60 LOC)
export function useBarcodeScan(
  onScan: (code: string) => void,
  opts: { minChars?: number; avgTimeMs?: number } = {},
) {
  // buffer ký tự; reset nếu > 100ms gap; flush khi Enter/Tab
}
```

### Phase 10 — RBAC per-user override (Y/c 10) — P3 — 6h

**Current state:**
- RBAC matrix `packages/shared/src/rbac/matrix.ts` (142 LOC) — 4 role × 13 entity × 6 action hardcoded.
- `can()` function (44 LOC) — pure role-based, không có override layer.
- `apps/web/src/lib/role-guard.ts` + `apps/web/src/server/session.ts` wrapper.

**Tasks:**

1. **Migration 0009e (0.5h):**
   ```sql
   CREATE TABLE app.user_permission_override (
     id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
     user_id uuid NOT NULL REFERENCES app.user_account(id) ON DELETE CASCADE,
     entity varchar(32) NOT NULL,
     action varchar(16) NOT NULL,
     granted boolean NOT NULL,  -- true=grant extra, false=revoke
     note text,
     created_by uuid REFERENCES app.user_account(id),
     created_at timestamptz DEFAULT now(),
     UNIQUE (user_id, entity, action)
   );
   CREATE INDEX ON app.user_permission_override(user_id);
   ```

2. **Update `can()` function (1h):**
   - Signature mới: `can(roles, action, entity, overrides?)`.
   - Logic: `roleGrant = any(role matrix) || false; override = overrides?.find(...); return override?.granted ?? roleGrant;`.
   - Backward compatible: overrides undefined → behave cũ.

3. **Session: load overrides (1h):**
   - `server/session.ts` `requireCan`: fetch `user_permission_override WHERE user_id = session.userId` + cache 5min.
   - Pass overrides vào `can()`.

4. **API (1h):**
   - `GET /api/admin/users/[id]/permissions` → trả:
     ```ts
     { roleMatrix: RbacMatrix, userRoles: Role[], overrides: Override[], effective: Record<entity, Record<action, boolean>> }
     ```
   - `PATCH /api/admin/users/[id]/permissions` body `{ entity, action, granted: boolean | null }`; `null` = delete override (fallback về role default).

5. **UI (2h):**
   - Trong `/admin/users/[id]` thêm tab "Quyền".
   - Matrix render 13 entity × 6 action, mỗi ô checkbox 3-state:
     - `default` (inherit role): xám, có icon lock.
     - `granted` override: xanh check.
     - `revoked` override: đỏ X.
   - Click ô → cycle default → granted → revoked → default.
   - Legend + color code + "Reset to role default" button per row.
   - Audit log per change.

6. **Audit (0.5h):**
   - Write `audit_event(action="PERMISSION_OVERRIDE_SET", entity="user", objectId=userId, notes="entity.action = granted|revoked|default")`.

## Risks & Mitigations

| # | Risk | Impact | Mitigation |
|---|---|---|---|
| 1 | Phase 4 (WO detail) 10-12h có thể trượt do scope lớn | Delay launch B | Tách 4.1–4.5 ship incremental; 4.1+4.2 ship trước để có routing thật |

| 3 | Seed demo PO 15 records có thể conflict với existing data | Duplicate PO | Idempotent seed: check `poNo` unique trước insert |
| 4 | Migration 0009e (user_permission_override) lỗi nếu đang có session active | Login thất bại | Deploy trong maintenance window; test rollback script |

| 6 | Phase 9 export xlsx có thể OOM với PO > 1000 | Server crash | Stream write với `exceljs` workbook stream; pagination export 500/batch |
| 7 | RBAC override check mỗi request thêm 1 query → latency | +~5ms/req | Cache 5min per-user memory LRU; invalidate on PATCH |
| 8 | ProgressCell breakdown query join 3-4 bảng có thể slow | Grid lag | Materialized view refresh 5min; hoặc cache React Query 60s |
| 9 | Phase 3 (Order Sản xuất editable) có thể đụng versionLock race | Lost update | Dùng optimistic lock versionLock cũ; retry logic FE |
| 10 | 4 pre-existing TS errors trong `purchaseOrders.ts` | Block CI nếu strict | Fix trong Phase 9 task 0 trước khi extend |
